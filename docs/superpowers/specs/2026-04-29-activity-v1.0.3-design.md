# Activity v1.0.3 Design (Note attachment fix)

- **作成日**: 2026-04-29
- **対象**: PlantUMLAssist v1.0.3 — v1.0.1 mid-insert note の attachment bug fix
- **base ブランチ**: `feat/activity-v1.0.2` (PR #13 merge 後は master 相当)
- **次ブランチ**: `feat/activity-v1.0.3`
- **親 spec**: `docs/superpowers/specs/2026-04-28-activity-v1.0.1-design.md` (note insertion 機能の出所)

## 1. バグ

### 症状
ユーザ報告: 「複雑なアクティビティ図を書いたときに Note を挿入すると選択したものとは違う Action に Note が付きます」

### 再現手順
1. mid-insert modal を開いて kind=note 選択
2. クリック位置によっては `resolveInsertLine` が `{line: N, position: 'before'}` を返す (action の上をクリックしたケース)
3. `addNoteAtLine(text, N, 'before', fields)` が呼ばれる
4. note 行が **target line N の前** に挿入される
5. PlantUML は note を直前 statement に attach するため、 **action at line N-1** に note が付く
6. ユーザの視覚的意図 (= action at line N) と attachment 結果が不一致

### 根本原因 (`src/modules/activity.js:653-666`)

```javascript
function addNoteAtLine(text, lineNum, position, fields) {
  var lines = text.split('\n');
  var targetIdx = position === 'before' ? lineNum - 1 : lineNum;  // ← bug
  // ...
}
```

PlantUML Activity の note は概念的に side-attachment であり、 DSL 上は **直前 statement** に必ず attach される。 「前/後」の position 引数は他 kind (action / control) では位置決め意義があるが、 note kind では attachment target を意図せずズラす副作用となる。

## 2. 修正

### scope (in)

- `addNoteAtLine` の `targetIdx` を **常に lineNum** にする (= position に関わらず target line の AFTER に挿入)
- API signature `(text, lineNum, position, fields)` は v1.0.1 互換のため維持。 position 引数は no-op に降格、 コメントで明示。
- Unit テスト追加:
  - 既存 2 テスト (位置を `'after'` で叩いていた) は据え置き、 動作確認
  - 新規 1 テスト: `position='before'` 渡しでも target line の AFTER に挿入される回帰テスト

### scope (out)

- property panel の `+ Note` ボタン (`addNote(node.endLine, ...)` 経由) — endLine 後挿入で attachment 正常、 ただし composite 内 indent 継承なし (cosmetic) → 別 sprint
- `note right of ALIAS` 構文への移行 — v1.x+ で別途検討
- E2E UC-9b (click 上方向 → 正しい attachment) — buildOverlay の composite 内 rect 制限により skip フォールバック必至、 unit テストで algorithm 担保のため省略

## 3. アーキテクチャ

### `src/modules/activity.js` の変更

```javascript
function addNoteAtLine(text, lineNum, position, fields) {
  var lines = text.split('\n');
  // PlantUML の note は直前 statement に attach されるため、 position 引数は
  // 概念的に意味を持たない (attach target は lineNum 自身であり、 その前後ではない)。
  // 互換性のため引数は維持するが内部では常に lineNum の after に挿入する。
  var targetIdx = lineNum;
  if (targetIdx < 0) targetIdx = 0;
  if (targetIdx > lines.length) targetIdx = lines.length;
  var indent = _resolveInsertIndent(lines, Math.min(targetIdx, lines.length - 1));
  fields = fields || {};
  var formatted = fmtNote(fields.position || 'right', fields.text || '');
  var newLines = Array.isArray(formatted) ? formatted : [formatted];
  var indented = newLines.map(function(l) { return indent + l; });
  var args = [targetIdx, 0].concat(indented);
  Array.prototype.splice.apply(lines, args);
  return lines.join('\n');
}
```

差分は 3 行 (`var targetIdx = ...` の置き換え + コメント追加)。

### `tests/activity-updater.test.js` の追加

`activity addNoteAtLine` describe (既存 2 tests) に 1 件追加:

```javascript
test('inserts after target line regardless of position arg (note attachment fix)', function() {
  var t = ':A;\n:B;\n:C;';
  // position='before' でも target line (line 2 = :B;) の AFTER に挿入される
  var out = actMod.addNoteAtLine(t, 2, 'before', { position: 'right', text: 'attach to B' });
  var lines = out.split('\n');
  // Expected order: :A; → :B; → note → :C;
  var aIdx = -1, bIdx = -1, noteIdx = -1, cIdx = -1;
  for (var i = 0; i < lines.length; i++) {
    if (lines[i] === ':A;') aIdx = i;
    if (lines[i] === ':B;') bIdx = i;
    if (lines[i].indexOf('note right') >= 0) noteIdx = i;
    if (lines[i] === ':C;') cIdx = i;
  }
  expect(aIdx).toBe(0);
  expect(bIdx).toBe(1);
  expect(noteIdx).toBe(2);  // note is RIGHT AFTER :B;, not before
  expect(cIdx).toBe(3);
});
```
