## 1. 背景と動機

v0.6.0 で Class diagram の 4 element kinds + 6 relations + members + nesting が overlay-driven で完成。ただし以下が繰越済:

- **Member 個別 SVG クリック選択**: 現状 property panel のリストでしか member を選べない。クラスが大きくなると目的の member を探すのに手数がかかる。
- **Note on class**: PlantUML で OO 設計の意図 / 実装メモ / TODO を class 横に置く頻出ユースケース。現状は DSL 直書きしか手段がない。

v0.6.1 のゴール:

1. **Member 個別クリック選択** — class box 内の attribute / method / enum-value 行を SVG 上で直接 click して編集できる
2. **Note on class** — `note (left|right|top|bottom) of NAME` 形式 (1行 + 複数行) を parse / render / property panel から作成・編集・削除できる
3. **`extractMultiLineTextBBoxes` API** を `core/overlay-builder.js` に追加 (Activity v0.7.0 で共有利用予定)

**v0.6.0 spec § 9 で Tier2 扱いだった「内部クラス」は v0.6.1 でも対象外** (本 polish では UX gap が大きい member click + note を優先、内部クラスは Tier2 もしくは別スプリント)。

## 2. 適用範囲 (in / out)

### v0.6.1 で達成する (in)

#### Member 個別クリック
- Class / interface / abstract class の attribute / method 行を SVG クリックで個別選択
- Enum の value 行を SVG クリックで個別選択
- 選択時の property panel: 既存 `_renderElementEdit` の class 全体表示 + 対象 member 行を inline edit fields 展開 + auto-scroll
- Shift+click 時: member は親 class に coerce、multi-select connect は class レベルでのみ動作
- `extractMultiLineTextBBoxes(g)` を `core/overlay-builder.js` に追加 (戻り値: `[{text, bbox, lineIndex, kind?}]`)

#### Note on class
- DSL: `note (left|right|top|bottom) of CLASS_ID : single-line` 形式
- DSL: 複数行 `note (left|right|top|bottom) of CLASS_ID\n  multi line\nend note` 形式
- 適用 element kind: class / interface / abstract / enum
- `parse()` で `notes[]` 配列を返す (新規)
- `addNote()` / `updateNote()` / `deleteNote()` operations
- 入口 (両方):
  - **A**: tail-add kind selector に `note` option 追加 (target class selector + 4-dir position + textarea)
  - **B**: 選択中 class panel に `+ Note` expander 追加 (position + textarea、target は選択中 class auto-fill)
- buildOverlay で note rect を生成 (`data-type=note` で selectable)
- Click 選択時の panel: position 4-dir selector + textarea + delete button (target は read-only label)
- Class 削除時: 紐付く note は cascade 削除 (orphan を残さない)

### v0.6.1 では やらない (out)

- **内部クラス (nested class)** — Tier2
- **Floating note** (`note "..." as N` + link) — Tier2 / v0.7.0+
- **Note on relation** (`note on link`) — Tier2
- **Member の drag-to-reorder** — v0.7.0+ で Activity の swim lane drag と統合設計
- **Member 同士 / member↔class connect** — PlantUML 仕様外

## 4. データモデル拡張

### 4.1 Note model (新規)

```javascript
{
  kind: 'note',
  id: '__n_0',                           // module 内 unique
  position: 'left' | 'right' | 'top' | 'bottom',
  targetId: 'Foo',                       // class/interface/abstract/enum の id
  text: 'first line\nsecond line',       // \n join (複数行も 1 string)
  line: 12,                              // note 開始行
  endLine: 14,                           // 1行 note なら == line、複数行なら end note 行
}
```

`parse()` の戻り値に `notes: []` を追加。

### 4.2 Selection state 拡張 (member type)

```javascript
// Member click 時の selection
{
  type: 'member',
  parentId: 'Foo',                       // 所属 class id
  memberIndex: 2,                        // class.members[] のインデックス
  kind: 'attribute' | 'method' | 'enum-value',
  // 互換のため id を合成 (selection-router で multi-select 集約用)
  id: 'Foo::__m_2',
}

// Note click 時の selection
{
  type: 'note',
  id: '__n_0',
  noteId: '__n_0',
  targetId: 'Foo',
}
```

### 4.3 capabilities 変更なし

`overlaySelection: true` / `multiSelectConnect: true` のまま。member coerce は selection-router 内部で完結。

## 5. ADR-108 — Note canonical form

PlantUML note の表記揺れを canonical 化:

| バリエーション | canonical |
|---|---|
| `note left : text` (target 省略) | **不採用** (parser は受理しない、target 明示必須) |
| `note left of Foo : text` (1行) | `note left of Foo : text` (空白固定: `note SP pos SP of SP id SP : SP text`) |
| `note left of Foo`<br/>`text`<br/>`end note` (複数行) | block 形式維持 (text 内改行ありなら自動 block化、なしなら 1行) |
| `note top of Foo` / `note bottom of Foo` | 同上 |
| Position 大文字小文字揺れ (`Left` / `LEFT`) | 全部小文字に正規化 |

emit ルール: text に `\n` を含むか改行 char count > 60 で複数行 (block) 形式、それ以外 1行。

ADR-108 として `docs/adr/ADR-108-note-canonical-form.md` に記載。

## 7. Overlay 設計

### 7.1 `extractMultiLineTextBBoxes(g)` (新規 API)

`core/overlay-builder.js` に追加。`<g class="entity">` 内の 1 つ以上の `<text>` 要素の bbox を行ごとに取得。

```javascript
/**
 * Extract per-line bboxes from an entity group's text content.
 * For class members: each <text> in <g class="entity"> is one member line.
 * For Activity actions (v0.7.0): each <tspan> may be a logical line.
 *
 * @param {Element} g - SVG <g class="entity"> or compatible
 * @param {object} [opts] - { mode: 'text-per-line' (default) | 'tspan-per-line' }
 * @returns {Array<{text, bbox, lineIndex}>}
 */
function extractMultiLineTextBBoxes(g, opts) {
  var mode = (opts && opts.mode) || 'text-per-line';
  var nodes = mode === 'tspan-per-line'
    ? g.querySelectorAll('text tspan')
    : g.querySelectorAll('text');
  var lines = [];
  for (var i = 0; i < nodes.length; i++) {
    var n = nodes[i];
    var bb = null;
    if (typeof n.getBBox === 'function') {
      try { bb = n.getBBox(); } catch (e) { /* jsdom fallback */ }
    }
    if (!bb) {
      bb = {
        x: parseFloat(n.getAttribute('x')) || 0,
        y: parseFloat(n.getAttribute('y')) || 0,
        width: 0,
        height: 14,
      };
    }
    lines.push({ text: (n.textContent || '').trim(), bbox: bb, lineIndex: i });
  }
  return lines;
}
```

class 用: `mode='text-per-line'` で member ごとに `<text>` が来る前提 (PlantUML SVG 標準)。
Activity 用 (v0.7.0): action label が複数行 = 1 `<text>` + 複数 `<tspan>` であれば `mode='tspan-per-line'` に切替。

### 7.2 class.buildOverlay 拡張

既存 entity rect に加え:

1. **Member rect**: entity 内の text bbox を `extractMultiLineTextBBoxes` で取得し、最初の N 行 (= class header / generics / stereotype) を skip して member 部分を抽出。`members[i] ↔ memberLines[i]` 順序対応。
2. **Note rect**: PlantUML note の SVG は座標ベースの polygon (折り角つき 5 点矩形) + text で描画される (`<g class="..."` や `data-qualified-name` は付かない)。`buildOverlay` は **note 形状の polygon を文書順でマッチング** して overlay rect を生成する:
   - SVG 直下を walk し `<polygon>` で頂点数 5 (折り角矩形) のものを文書順に列挙
   - parse 結果の `notes[]` 配列と順番対応 (PlantUML は DSL 順 = SVG 順)
   - 各 polygon の bbox を `extractBBox`-相当ロジックで取得し、`data-type=note data-id=__n_N` rect を overlay に追加
   - マッチ失敗時 (polygon 形状判定が外れる skin) は warn を出して該当 note の overlay rect を skip。当該 note は property panel の class detail から `notes` セクション経由で選択可能 (フォールバック)

実装メモ:
- Class header の `<text>` (class名 / `<<stereo>>` / generics) は member より先に来る。skip 数は `class.label / class.stereotype / class.generics` の有無から計算 (typically 1〜3 行 skip)
- 順序が崩れる corner case: stereotype が icon (PlantUML が `<<entity>>` を database/circle icon に置換するスキン設定) の場合 text 行ではなく `<image>` になる。当面は default skin 前提でテスト。

### 7.3 selection-router 拡張

```javascript
// shift+click で multi-select 集約時
function _coerceToParent(selection) {
  return selection.map(function(s) {
    if (s.type === 'member') {
      return { type: s.parentKind || 'class', id: s.parentId };
    }
    return s;
  });
}
```

multi-select connect は既存の `class/interface/abstract/enum` のみ受理ロジックなので、coerce 後の selection でそのまま動作。

## 9. リスクと対策

| リスク | 影響 | 対策 |
|---|---|---|
| **Note の SVG polygon マッチング不安定** | overlay click 不可 | 文書順 + 5点 polygon の二重判定。マッチ失敗時は warn + 該当 note の overlay rect skip。フォールバックとして親 class 選択時の panel 内 `notes` リストから edit 可能にする |
| **Member 行と SVG `<text>` の順序ズレ** | wrong member が選択される | header skip 数の自動計算 + 各 line の text 内容を model member の `visibility + name` と照合 (照合失敗で warn) |
| **複数行 note の `end note` parse fragility** | 中途半端な block で parser が固まる | open / close stack で block scope を厳密管理 (class block と同じパターン)、未閉 block は warn |
| **Member click が class click と干渉** | クラス全体 select したい時に member が選ばれる | class header (label 行) は member rect 生成対象外、空白部分は class rect でカバー (既存 entity bbox)。z-index で member rect を class rect の上 |
| **`extractMultiLineTextBBoxes` の API 仕様が Activity と齟齬** | v0.7.0 で再設計コスト | opts.mode で切替可能、戻り値 schema は最小化 (text + bbox + lineIndex のみ、kind は呼び出し側で付与) |
| **PlantUML が note を `class .. note_id` の link 形式で render** | 1行 directional note と SVG 構造が違う | v0.6.1 では directional 形式のみサポート、`as N` floating は明示的に out of scope |
