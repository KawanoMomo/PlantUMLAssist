## コンテキスト

PlantUML の GUI 編集ツールを新規構築。MermaidAssist の設計パターンを流用しつつ、Java 依存をバックエンドで吸収する。

## 対策

1. **Python stdlib バックエンド** (ADR-101): server.py で静的配信 + /render endpoint
2. **local/online デュアルモード** (ADR-103): `<select>` + `localStorage` 永続化
3. **非同期 render pipeline** (ADR-102): debounce 150ms + status indicator
4. **Sequence Diagram モジュール**: DiagramModule v2 踏襲、8 participant types + 10 arrows
5. **MermaidAssist core/ui 流用**: html-utils, text-updater, history, selection, connection-mode, properties
6. **Tab インデント** (workspace ADR-011): 2 スペース indent/outdent

## 教訓

- MermaidAssist の core/ui 流用は効率的。図形モジュールのみ新規実装で済む
- selection/history の init API は MermaidAssist 固有 (`callback`, `{getMmdText/setMmdText/onUpdate}`) なので spec 記載時は実装を確認
- Java 未インストール環境でも UI 側は動作、online 切替で代替可能
